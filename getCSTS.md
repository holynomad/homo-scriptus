  7장. 테스트 자동화
    - 7.1 개요
      - 지원 업무별 테스트 도구 분류
        - 테스트 관리
        - 정적 테스트
        - 동적 테스트
        - 테스트 실행 및 로깅
        - 성능 및 모니터링
      - 셀레니움을 이용한 JUnit Webdriver 테스트
        - 예) 부하 테스트 또는 스트레스 테스트 위해 100명의 사용자 동시 시스템 접근, 어떻게 ?
        - 자동화된 테스트 스크립트를 100개의 스레드나 프로세스를 이용하여 자동으로 수행한다면 훨씬 효과적으로 테스트 수행 가능

    - 7.2 테스트 자동화 분야 및 테스트 도구
      - 완성도 높은 자동화 인프라는 테스트의 여러 단계에 걸쳐 전체적인 테스트 활동의 전문성과 안정성에 관여함 : 키스스토비 & 마크버그먼의 SEARCH 모델
      - Setup, Execution, Analysis, Report, Clean-up, Help의 총 6단계
      - ISO/IEC/IEEE 29119 : 조직 테스트 프로세스, 테스트 관리 프로세스, 동적 테스트 프로세스로 분류 (각 단계 모두 자동화 가능)
        - 테스트 설계 도구 : 테스트 케이스를 생성하는 도구
        - 테스트 환경 구축 도구 : 
          - IaC : '인프라 코드화', 시스템 환경을 수동으로 구성하는 대신 시스템 환경 구성 정보를 기록한 스크립트를 사용하여 자동으로 인프라 구성
          - 도커 : IaC 개념을 바탕으로 애플리케이션 실행 환경을 자동으로 설치할 수 있는 도구
            - 소프트웨어 컨테이너 라는 단위로 패키징, 애플리케이션을 신속 배포 및 확장할 수 있고 코드가 문제없이 개발자 플랫폼에 구애받지 않고 실행됨
        - 테스트 실행 도구 : 테스트 케이스를 자동으로 실행할 수 있도록 스크립트로 변환하여 실행 (record & playback 참고)
          - 선형(linear) 테스트 실행 프레임워크
          - 모듈(modular based) 테스트 실행 프레임워크
          - 데이터 주도(data-driven) 테스트 실행 프레임워크
          - 키워드 주도(keyword-driven) 테스트 실행 프레임워크
      - 이슈 관리 도구
        - 결함 생명주기 (defect life cycle) 관리기능
    - 7.3 테스트 도구 선정
      - 테스트 자동화는 매뉴얼 테스트를 대치하기 보단 상호보완 방향으로 가야함
