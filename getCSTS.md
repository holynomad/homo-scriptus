## 2024-05-19

  ```
  1일차

  1장. 테스트 개요
    - 1.1 테스트 목적 : 결함 검출, 품질 평가, 프로세스 개선
    - 1.2 오류, 결함, 장애
      - 오류 : 결함을 생기게 한 개발자의 행위
      - 결함 : 소프트웨어 내 장애를 유발할 수 있는 문제
        - 3가지 유형 : 누락, 부정확한 구현, 비관련
          - 누락 : 요구 명세에 명시된 요구사항이 시스템의 구현에 반영되지 않은 결함
          - 부정확한 구현 : 요구 명세에 명시된 요구사항이 소프트웨어에 부정확하게 반영된 결함
          - 비관련 : 요구 명세와 관련되지 않은 구현
      - 장애 : 소프트웨어 요구사항과 다르게 동작
    - 1.3 테스트의 현실/실제
    - 1.4 테스트와 품질
      - ISO 25010 : 소프트웨어 품질 유형 8가지
        - 기능 적합성
        - 성능 효율성
        - 호환성
        - 사용성
        - 신뢰성
        - 보안성
        - 유지보수성
        - 이식성
      - 테스트, V&V, 품질보증 포함관계
        - 테스트 < V&V < 품질보증
    - 1.5 테스트 기본 용어
      - 컴포넌트(단위) 테스트
      - 통합 테스트
      - 테스트 설계기법
        - 정적 테스트 : 테스트 대상을 실행하지 않고 테스트 수행하는 방식 (리뷰, 정적분석)
        - 동적 테스트 : 소프트웨어를 실행하는 방식으로 테스트를 수행하여 결함 검출 (명세 기반 방법, 구조 기반 방법, 경험 기반 방법)
  ```

  굄둥이들 운동회 출동으로 분주하고 알찼던 하루.

  3장. 소프트웨어 개발 단계와 테스트
    - 3.1 개요
      - 테스트 유형기준
      - 테스트 레벨기준
      - 테스트 설계기준
    - 3.2 컴포넌트 테스트
      - 개별적인 모듈(또는 컴포넌트)의 테스트를 말하며, 구현 단계에서 각 모듈을 구현한 후에 수행 (TDD처럼 코드 개발 전 테스트 케이스 먼저 생성도 가능)
      - 테스트 환경 주요 구성요소 : 테스트 드라이버(driver), 테스트 스텁(stub)
        - 모의(mock) 객체는 스텁의 객체 지향 버전
        - 예시) 모의 객체를 이용한 JUnit 테스트 케이스
          - 더미 객체, 테스트 스텁, 테스트 스파이, 가짜 객체
      - FIRST 원칙 : Fast(빠르게), Isolated(테스트간 의존하지 않도록), Repeatable(동일한 결과 반복되도록), Self-Validating(사람의 개입 없이 테스트 통과되었는
                    지 알 수 있도록 작성), Timely(테스트 대상이 되는 코드가 작성되는 시점에 수행)
    - 3.3 통합 테스트
      - 컴포넌트를 통합하는 과정에서 수행되는 테스트
      - 상호작용 초점, 기능 초점
      - 빅뱅 방식 : 통합 대상 컴포넌트가 많은 경우, 전체 컴포넌트를 한 번에 통합하여 테스트하는 방법
      - 점진적(Incremental) 방식 : 적은 수의 컴포넌트를 차례로 통합하는 방식
        - 상향식 통합 테스트
        - 하향식 통합 테스트
        - 샌드위치 통합 테스트
    - 3.4 시스템 테스트 및 인수 테스트
      - 시스템 테스트 : 통합테스트 완료 후 전체 시스템이 시스템 명세에 따라 개발되었는지 검증하기 위해 수행하는 테스트
      - 인수 테스트 : 실제 사용자 요구사항 만족하는지 확인하기 위한 테스트
    - 3.5 리그레션 테스트
      - 유지보수 단계에서 SW가 수정된 후에 변경이 올바르게 되었는지, 변경 작업으로 인해 새로운 결함이 유발되었는지 검사하기 위해 수행하는 테스트
        - 유지보수 종류 : 결함 수정, 기능 보강, 적응, 예방
        - 종류 : retest-all, 선택적 리그레션 테스트, 테스트 최소화, 테스트 우선순위화
          - 선택적 리그레션 테스트 : 기존 테스트 케이스 중 일부만 선정
          - 테스트 최소화 : 중복된 테스트 케이스 제거
          - 테스트 우선순위화 : 우선순위 높은 테스트 케이스만 활용
            - APFD (Average Percentage of Faults Detected) : 우선순위 효과성 평가 위한 척도로 많이 사용
